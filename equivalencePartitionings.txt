Summaries
  inputDomainSummary
    - usersRegisterInputs: email, password, username, firstName, lastName, plus uniqueness constraints
    - usersLoginInputs: email, password, plus accountState constraints
    - usersRoleAssignmentInputs: contextRole, username, roleName
    - projectsCreateInputs: contextRole, name, repositoryID, description
    - projectsUpdateInputs: contextRole, projectID, optionalName, optionalDescription, optionalRepositoryID
    - projectsDeleteInputs: contextRole, projectID
    - projectsAddUserInputs: contextRole, projectID, userID, linkState
    - projectsRemoveUserInputs: contextRole, projectID, userID, linkState
  outputDomainSummary
    - successOutputs: authTokenAndUser, projectEntityOrBoolean, updatedEntities
    - errorOutputs: validationErrors, notAuthorizedErrors, notFoundErrors, conflictErrors

Category Partitioning
  note: test cases are derived by combining exactly one choice from each category. One-valid-at-a-time means one category is invalid while all others are valid.
  prunedCategories
    - descriptionCategory: validAndMissingMerged (both map to success)
    - repositoryCategory: repositoryMissingMergedWithRepositoryValid (both map to success)
  finalCategories
    - register: emailCategory x passwordCategory x usernameCategory x firstNameCategory x lastNameCategory x uniquenessCategory
    - login: emailCategory x passwordCategory x accountStateCategory
    - roleAssignment: contextCategory x userCategory x roleCategory
    - createProject: contextCategory x nameCategory x repositoryCategory x descriptionCategory
    - updateProject: contextCategory x projectIdCategory x nameCategory x descriptionCategory x repositoryCategory
    - deleteProject: contextCategory x projectIdCategory
    - addUserToProject: contextCategory x projectIdCategory x userIdCategory x linkStateCategory
    - removeUserFromProject: contextCategory x linkStateCategory
